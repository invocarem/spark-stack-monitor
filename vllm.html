Isolated from the SGLang Stack Monitor. Host Docker only; same API (/api/…) as the main app.

Container vllm-node-tf5

Starts an idle stack (sleep infinity as PID 1), like the SGLang monitor. Then open Tools below and run vLLM: launch serve.sh (detached) — it runs repo file scripts/vllm/serve.sh (mounts at /workspace/scripts/vllm/serve.sh). Edit that script to try different vllm serve flags. Image: MONITOR_VLLM_TF5_IMAGE or vllm-node-tf5:latest. Port MONITOR_STACK_HOST_PORT→8000.

Docker logs (PID 1)

PID 1 is sleep, so this stays quiet. Server output goes to /workspace/.monitor/vllm-launch.log (see the launch tool message) — tail that file in the container if needed.
