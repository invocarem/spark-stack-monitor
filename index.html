SGLang Stack Monitor

About this monitor

Container runs the stack image on the host (docker run / start / stop). Launch runs scripts/*.sh inside that container to start sglang.launch_server. Optional vLLM testing is a separate page (does not change presets here). Set the model id for Chat/Benchmark; other tabs: Logs, Tools, metrics, benchmark, chat.

Container Control

Starts/stops the stack container on the host. Presets mirror containers/run-docker*.sh and run detached (sleep infinity) so the Launch tab can execute scripts/*.sh inside it.

Host env and paths

API runs on host Docker. Optional HF_TOKEN is passed through. Mount root is MONITOR_REPO_ROOT (or auto-detected). Host port MONITOR_STACK_HOST_PORT (default 8000) maps to container 8000.

Tip: keep it running, then use the Launch tab.

Launch SGLang (launch_server)

After a stack container is running, pick it here and run a script from ./scripts to start the LLM with python3 -m sglang.launch_server. While the server is running, starting again is blocked. Script output while the model loads is on the Logs tab (launch script log).

How it works

Host repo ./scripts is mounted at /workspace/scripts. The API runs docker exec -d … with the script’s stdout/stderr appended to /workspace/.monitor/sglang-launch.log (that output does not appear in docker logs, which is only the container’s main process). Open the Logs tab to tail that file; Check launch server and SGLang metrics show readiness.

Launch args from script (python3 -m sglang.launch_server): edit values or uncheck to remove unsupported flags.

Select a script to load args.

Chat & Benchmark model id

Must match --served-model-name from your script. Saved in this browser; Check launch server syncs it to the running server’s model id when they differ.

Container logs

Prefer Launch script log (/workspace/.monitor/sglang-launch.log): that is where Launch tab scripts run (via docker exec), including load progress. Monitor stack containers use sleep infinity as PID 1, so Docker logs (same as host docker logs) is usually almost empty—not your LLM session.
