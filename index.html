Spark Stack Monitor

About this monitor

Container runs the stack image on the host (docker run / start / stop). Launch runs scripts/sglang/*.sh inside that container to start sglang.launch_server or sglang serve. Optional vLLM testing is a separate page (does not change presets here). Set the model id for Chat/Benchmark; other tabs: Logs, Tools, metrics, benchmark, chat.

Container Control

Starts/stops the stack container on the host. Presets mirror containers/sglang/run-docker*.sh and run detached (sleep infinity) so the Launch tab can execute scripts/sglang/*.sh inside it.

Host env and paths

API runs on host Docker. Optional HF_TOKEN is passed through. Mount root is MONITOR_REPO_ROOT (or auto-detected). SGLang presets publish MONITOR_STACK_HOST_PORT (default 30000) to container port 30000 (same as scripts/sglang/*.sh). vLLM presets default 8000→8000.

Launch workflow (SGLang)

Same idle container either way; this syncs the Launch tab “Multi-node cluster” option (exports / rank / dist). If MONITOR_CLUSTER_APPLY is set in .env, this block is hidden and Launch follows .env only. Otherwise use Single for one machine or Cluster when using --nnodes > 1 across hosts.

Tip: keep it running, then use the Launch tab.

Launch SGLang (launch_server / sglang serve)

After a stack container is running, pick it here and run a script from ./scripts/sglang to start the LLM with python3 -m sglang.launch_server or sglang serve. While the server is running, starting again is blocked. Script output while the model loads is on the Logs tab (launch script log).

How it works

Host repo ./scripts/sglang is mounted at /workspace/scripts/sglang. The API runs docker exec -d … with the script’s stdout/stderr appended to /workspace/.monitor/sglang-launch.log (that output does not appear in docker logs, which is only the container’s main process). Open the Logs tab to tail that file; Check launch server and SGLang metrics show readiness.

Launch args from script (python3 -m sglang.launch_server or sglang serve): edit values or uncheck to remove unsupported flags.

Select a script to load args.
Multi-node cluster

Optional export lines before your launch script: NCCL / GLOO / MASTER_* (same for SGLang and vLLM—PyTorch / NCCL pick them up). Use different interfaces on each host; run Run script per node. Repo .env can prefill via the API when you use npm run dev. SGLang: merges --dist-init-addr / --nnodes / --node-rank into launch_server when missing from the script. vLLM: merges --master-addr / --master-port (from the MASTER_ADDR / MASTER_PORT fields), --nnodes, and --node-rank. Set --distributed-executor-backend (mp vs ray) in your script—see vLLM serve CLI (--dist-init-addr is SGLang-only).

Chat & Benchmark model id

Must match --served-model-name from your script. Saved in this browser; Check launch server syncs it to the running server’s model id when they differ.

Container logs

Prefer Launch script log (/workspace/.monitor/sglang-launch.log): that is where Launch tab scripts run (via docker exec), including load progress. Monitor stack containers use sleep infinity as PID 1, so Docker logs (same as host docker logs) is usually almost empty—not your LLM session.
